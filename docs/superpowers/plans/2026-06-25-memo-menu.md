# Tek Dürümcü Memo Digital Menu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a modern, highly responsive digital menu web app for "Tek Dürümcü Memo" supporting Turkish, English, and Russian languages, and featuring a PIN-protected client-side Admin Panel with full CRUD operations, local storage synchronization, JSON backup import/export, and dynamic QR Code generation.

**Architecture:** A static client-side Single Page Application (SPA) structured as a set of ES modules (`data.js`, `app.js`, `admin.js`) coupled with `index.html` and a utility stylesheet `style.css`.
* **State Management**: A global reactive state object that stores state in `localStorage` and triggers visual refreshes on mutate.
* **Database**: Browser `localStorage` holds the menu settings.
* **Styling**: Vanilla CSS utilizing CSS Custom Properties for theme tokens, custom grid layouts, and micro-animations.

**Tech Stack:** 
* HTML5 / CSS3 / ES6 Javascript Modules
* Node.js (v18+) built-in test runner (`node --test`) for unit tests
* `qrcode.js` (via CDN) for local QR code rendering

## Global Constraints
* No external CSS frameworks (no Tailwind unless explicitly configured). Use Vanilla CSS.
* Language selection defaults to Turkish (`tr`), with translations for English (`en`) and Russian (`ru`).
* If a translation field is blank or missing, automatically fallback to Turkish (`tr`).
* The application must load instantly and use visual micro-animations (transitions, scale-ups, lifts).
* Security code for Admin Panel is PIN `1234` by default.

  import test from 'node:test';
  import assert from 'node:assert';
  import { DEFAULT_MENU_DATA } from '../data.js';

  // Helper function to resolve translation with fallback
  function getTranslation(item, lang, field) {
    if (item.translations[lang] && item.translations[lang][field]) {
      return item.translations[lang][field];
    }
    return item.translations['tr'][field] || '';
  }

  test('Translation fallback returns target language or defaults to Turkish', () => {
    const item = {
      translations: {
        tr: { name: 'Adana', description: 'Acılı' },
        en: { name: 'Adana Kebab' } // Missing description
      }
    };
    
    assert.strictEqual(getTranslation(item, 'en', 'name'), 'Adana Kebab');
    assert.strictEqual(getTranslation(item, 'en', 'description'), 'Acılı');
    assert.strictEqual(getTranslation(item, 'ru', 'name'), 'Adana');

  Write logic to initialize state from localStorage, and load `DEFAULT_MENU_DATA` if missing.
  ```javascript
  import { DEFAULT_MENU_DATA } from './data.js';

  let menuState = JSON.parse(localStorage.getItem('memo_menu_data'));
  if (!menuState) {
    menuState = DEFAULT_MENU_DATA;
    localStorage.setItem('memo_menu_data', JSON.stringify(menuState));
  }
  ```

- [ ] **Step 2: Dynamic Opening Status Calculator**
  * Restaurant Hours: Opens: 08:00, Closes: 04:00 (next day).
  * Write a function `checkIsOpen()` that checks current local time and returns true/false.
  ```javascript
  export function checkIsOpen(currentHour, currentMinute) {
    // CurrentHour is 0-23
    // Open hours: 08:00 - 04:00
    // We are closed between 04:00 and 08:00.
    if (currentHour >= 4 && currentHour < 8) {
      return false; // Closed
    }
    return true; // Open
  }
  ```
